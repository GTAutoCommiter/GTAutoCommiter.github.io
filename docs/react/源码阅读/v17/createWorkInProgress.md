### 目的

createWorkInProgress 的主要目的是为下一个渲染过程准备一个 fiber 节点。通过创建或重用一个工作进行中 fiber，React 可以高效地管理更新并确保 UI 正确渲染。

这个函数是 React 双缓冲策略的关键部分，其中维护了两个版本的 fiber 树：一个用于当前 UI，另一个用于即将到来的更改。这允许 React 以非阻塞的方式执行更新，从而提高性能和响应速度。
