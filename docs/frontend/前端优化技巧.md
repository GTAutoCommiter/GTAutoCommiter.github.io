# 请求相关

## 缩短遍历请求请求接口时长

1. 查看当前请求时间。

   * 在控制台对测试代码前后进行打断点

   * 编辑断点，通过`console.time(<sign>)`与`console.timeEnd(<sign>)`将需要获取请求时间

   * 控制台获取修改前执行时间，与修改后执行时间进行对比。

2. 对代码进行优化。

### 使用Promise.all()｜Promise.allSettled();

* 使用场景：
  1. 遍历请求同一个接口，只是参数不同。
  2. 前一请求的结果不影响后一请求的参数（如：初始化获取资源）。
